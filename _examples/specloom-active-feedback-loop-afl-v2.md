## 1. Overview

### 1.1 What This Feature Does

SpecLoom is a screen recording → requirements pipeline. Artifacts — spec documents, epic tickets, implementation guides, and knowledge base facts — are currently generated once and never corrected structurally. When the AI makes a mistake (misheard product name, fabricated requirement, wrong inference in the KB), the only options are to regenerate from scratch (losing context) or manually edit (overwritten on next regeneration).

The **Active Feedback Loop (AFL)** feature gives every rendered element in every artifact a ≤ 2-click flag affordance. Captured corrections are stored as **feedback notes** anchored to **stable element identities** that survive regeneration. Subsequent regenerations are **feedback-aware**: active notes are compiled into a structured `<corrections>` XML block injected into the generator prompt. A deterministic **verification pass** independently checks whether each correction was applied. The cycle closes: the same error class cannot re-enter on subsequent regenerations.

### 1.2 Core User Journey

A user spots "Eurobill" in a spec section (the model misheard "Oracle" from the demo video). They hover the section → click the flag → fill the dialog in two steps → submit. The note is captured. On the next KB rebuild, `applySourceCorrections` patches the transcript at read time (without mutating the transcript row). Follow-on spec regenerations receive a `<corrections>` block. After regeneration, the user reviews the diff — the section is highlighted blue with attribution to their note. They click Resolve. The promotion heuristic fires; they accept and the principle "Product name is Oracle, not Eurobill" enters every future regeneration prompt.

### 2.15 `afl_commit_regen_aux()` Stored Procedure

```sql
-- Migration: {{TS14}}_afl_commit_regen_aux_fn.sql
-- Atomic Window-B commit: element identity upserts, application records,
-- reconciliation suspicions, note-status updates. All-or-nothing.

CREATE OR REPLACE FUNCTION public.afl_commit_regen_aux(
  p_project_id                uuid,
  p_processing_job_id         uuid,
  p_artifact_kind             text,
  p_artifact_logical_id       text,
  p_artifact_version          integer,
  p_kb_version                integer,
  p_reconciled_elements       jsonb,  -- array of { element_key, element_type, parent_identity_id?, last_known_content, content_hash, element_identity_id }
  p_removed_identity_ids      uuid[],
  p_application_records       jsonb,  -- array of ApplicationRecord shape
  p_suspicions                jsonb,  -- array of ReconciliationSuspicion shape
  p_applied_ok_note_ids       uuid[]
)
RETURNS jsonb
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_tenant_id       uuid;
  v_element         jsonb;
  v_app_rec         jsonb;
  v_suspicion       jsonb;
  v_identities_up   integer := 0;
  v_apps_inserted   integer := 0;
  v_suspicions_ins  integer := 0;
  v_notes_updated   integer := 0;
  v_now             timestamptz := now();
BEGIN
  SELECT tenant_id INTO v_tenant_id FROM public.projects WHERE id = p_project_id;

  -- 1. Upsert reconciled element_identities
  FOR v_element IN SELECT * FROM jsonb_array_elements(p_reconciled_elements) LOOP
    INSERT INTO public.element_identities (
      id, project_id, tenant_id,
      artifact_kind, artifact_logical_id, element_key, element_type,
      parent_identity_id, last_known_content, last_known_content_hash,
      first_seen_at, last_seen_at, last_seen_version
    )
    VALUES (
      COALESCE((v_element->>'element_identity_id')::uuid, gen_random_uuid()),
      p_project_id, v_tenant_id,
      p_artifact_kind, p_artifact_logical_id,
      v_element->>'element_key', v_element->>'element_type',
      NULLIF(v_element->>'parent_identity_id','')::uuid,
      LEFT(COALESCE(v_element->>'last_known_content',''), 4000),
      v_element->>'content_hash',
      v_now, v_now, p_artifact_version
    )
    ON CONFLICT (project_id, artifact_kind, artifact_logical_id, element_key)
    DO UPDATE SET
      last_known_content = EXCLUDED.last_known_content,
      last_known_content_hash = EXCLUDED.last_known_content_hash,
      last_seen_at = v_now,
      last_seen_version = p_artifact_version,
      removed_at = NULL,
      removed_in_version = NULL,
      updated_at = v_now;
    IF FOUND THEN v_identities_up := v_identities_up + 1; END IF;
  END LOOP;

  -- 2. Mark removed identities (do NOT delete; orphaned notes may reference them)
  IF array_length(p_removed_identity_ids, 1) IS NOT NULL THEN
    UPDATE public.element_identities
       SET removed_at = v_now, removed_in_version = p_artifact_version, updated_at = v_now
     WHERE id = ANY(p_removed_identity_ids) AND project_id = p_project_id AND removed_at IS NULL;
  END IF;

  -- 3. Insert application records (idempotent via UNIQUE on (feedback_note_id, processing_job_id))
  FOR v_app_rec IN SELECT * FROM jsonb_array_elements(p_application_records) LOOP
    INSERT INTO public.feedback_note_applications (
      project_id, tenant_id, feedback_note_id, processing_job_id,
      artifact_kind, artifact_logical_id, artifact_version, kb_version,
      application_status, status_detail, affected_identity_ids,
      verified_by_pass, verification_notes, applied_at
    )
    VALUES (
      p_project_id, v_tenant_id,
      (v_app_rec->>'feedback_note_id')::uuid, p_processing_job_id,
      p_artifact_kind, p_artifact_logical_id, p_artifact_version, p_kb_version,
      (v_app_rec->>'application_status')::public.feedback_application_status,
      NULLIF(v_app_rec->>'status_detail',''),
      COALESCE(ARRAY(SELECT jsonb_array_elements_text(v_app_rec->'affected_identity_ids'))::uuid[], '{}'::uuid[]),
      COALESCE((v_app_rec->>'verified_by_pass')::boolean, false),
      NULLIF(v_app_rec->>'verification_notes',''),
      v_now
    )
    ON CONFLICT (feedback_note_id, processing_job_id) DO NOTHING;
    IF FOUND THEN v_apps_inserted := v_apps_inserted + 1; END IF;
  END LOOP;

  -- 4. Insert reconciliation suspicions
  FOR v_suspicion IN SELECT * FROM jsonb_array_elements(p_suspicions) LOOP
    INSERT INTO public.reconciliation_suspicions (
      project_id, tenant_id, processing_job_id,
      artifact_kind, artifact_logical_id,
      suspicion_kind, prior_identity_id, new_identity_id,
      similarity, orphaned_note_ids, detail, resolution
    )
    VALUES (
      p_project_id, v_tenant_id, p_processing_job_id,
      p_artifact_kind, p_artifact_logical_id,
      (v_suspicion->>'suspicion_kind')::public.reconciliation_suspicion_kind,
      NULLIF(v_suspicion->>'prior_identity_id','')::uuid,
      NULLIF(v_suspicion->>'new_identity_id','')::uuid,
      NULLIF(v_suspicion->>'similarity','')::double precision,
      COALESCE(ARRAY(SELECT jsonb_array_elements_text(v_suspicion->'orphaned_note_ids'))::uuid[], '{}'::uuid[]),
      v_suspicion->>'detail',
      'pending'::public.reconciliation_suspicion_resolution
    );
    v_suspicions_ins := v_suspicions_ins + 1;
  END LOOP;

  -- 5. Mark applied_ok notes as 'applied' (guarded by status, idempotent)
  IF array_length(p_applied_ok_note_ids, 1) IS NOT NULL THEN
    UPDATE public.feedback_notes
       SET status = 'applied',
           applied_in_artifact_version = p_artifact_version,
           applied_in_kb_version = p_kb_version,
           updated_at = v_now
     WHERE id = ANY(p_applied_ok_note_ids)
       AND project_id = p_project_id
       AND status IN ('active', 'applied');
    GET DIAGNOSTICS v_notes_updated = ROW_COUNT;
  END IF;

  RETURN jsonb_build_object(
    'identities_upserted',   v_identities_up,
    'identities_removed',    COALESCE(array_length(p_removed_identity_ids, 1), 0),
    'applications_inserted', v_apps_inserted,
    'suspicions_inserted',   v_suspicions_ins,
    'notes_marked_applied',  v_notes_updated,
    'committed_at',          v_now
  );
END;
$$;

REVOKE ALL ON FUNCTION public.afl_commit_regen_aux(
  uuid, uuid, text, text, integer, integer, jsonb, uuid[], jsonb, jsonb, uuid[]
) FROM PUBLIC;
GRANT EXECUTE ON FUNCTION public.afl_commit_regen_aux(
  uuid, uuid, text, text, integer, integer, jsonb, uuid[], jsonb, jsonb, uuid[]
) TO service_role;
```

### 4.1 `FeedbackAnchor` — `src/components/feedback/FeedbackAnchor.tsx`

Wraps any renderable element. Adds the flag button + badge affordance. All state machines for identity minting and feedback loading are encapsulated here.

```tsx
export interface FeedbackAnchorProps {
  projectId: string;
  artifactKind: 'spec_document' | 'epic_ticket' | 'guide' | 'kb_fact';
  artifactLogicalId: string;
  elementKey: string;
  elementType: string;
  elementContent: string;           // snapshot used to prefill wrong_content
  children: React.ReactNode;

  // Provenance (optional — shown in dialog for source-layer toggle)
  sourceInputSourceId?: string;
  sourceVideoRange?: { start: number; end: number; videoId: string };

  // Visual variants
  density?: 'inline' | 'block';     // default: 'block'
  position?: 'tr' | 'tl' | 'br' | 'bl'; // default: 'tr'
  disabled?: boolean;               // feature flag off → true
}
```

**State machine:**

```
Mount
  → useElementIdentity() fires POST /mint-identity (idempotent)
  → identityId resolves (or stays null if project not enabled)
  → useFeedbackForScope(identityId) subscribes

Hover (group-hover CSS)
  → flag button fades in (opacity-0 → opacity-100)
  → if activeCount > 0: badge always visible (opacity-100)

Flag button click
  → opens FeedbackDialog with initialValues prefilled
  → on submit success: cache invalidated, badge updates

Disabled=true
  → renders children only, no flag, no badge
```

**CSS strategy:**

```tsx
<div className={cn("group relative", density === 'block' && "w-full")}>
  {children}
  {!disabled && (
    <>
      <button
        aria-label="Flag element for feedback"
        className={cn(
          "absolute z-10",
          position === 'tr' && "top-2 right-2",
          position === 'tl' && "top-2 left-2",
          position === 'br' && "bottom-2 right-2",
          position === 'bl' && "bottom-2 left-2",
          "opacity-0 group-hover:opacity-100 focus-visible:opacity-100",
          "@media (hover: none) { opacity-100 }",  // always visible on touch
          "transition-opacity duration-100",
          "h-8 w-8 rounded-md bg-white/90 dark:bg-zinc-800/90 shadow-sm",
          "hover:bg-white dark:hover:bg-zinc-800",
          "flex items-center justify-center",
          activeCount > 0 && "opacity-100"  // persist when feedback exists
        )}
      >
        <FlagIcon className="h-4 w-4" />
      </button>
      {(activeCount + appliedCount > 0) && (
        <FeedbackBadge identityId={identityId} activeCount={activeCount} appliedCount={appliedCount} />
      )}
    </>
  )}
</div>
```

**Edge-case positioning:**

| Container | Strategy |
|---|---|
| Table cell (`<td>`) | `density='inline'`; flag renders end-of-line after children |
| Inside popover/modal | Use React Portal to `document.body` for the dialog |
| `overflow-hidden` parent | Parent adds `overflow-visible` on hover-within, OR flag translates outside via `translate-x-full` |
| Very small element (height < 40px) | Fall back to `density='inline'` via ResizeObserver |

**Responsive:**
- Mobile (<640px): `FeedbackDialog` goes full-screen. Flag button always visible (`@media (hover: none)`)
- Desktop (>640px): hover-to-reveal behavior with flag at `position` corner

**Keyboard:** `Tab` focuses the flag button. `Enter`/`Space` opens dialog. `Escape` closes dialog (with confirmation if form dirty).

**States:**
- Default: children rendered; flag invisible until hover
- Hover/focus: flag fades in
- Has feedback: badge shows, flag always visible
- Disabled: children only, no flag affordance
- Identity resolving: flag renders but is disabled + spinner inside

## 5. Authentication & Authorization

### 5.1 RLS Policy Template (All 6 New AFL Tables)

The tenant-aware pattern verified from §3.4 applies to every new table:

```sql
-- Apply this 4-policy pattern to all 6 new AFL tables:
-- element_identities, feedback_notes, feedback_note_applications,
-- project_principles, source_corrections, reconciliation_suspicions

ALTER TABLE public.{table} ENABLE ROW LEVEL SECURITY;

-- SELECT: tenant-aware
CREATE POLICY {table}_select_own
  ON public.{table} FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM public.projects p
      WHERE p.id = {table}.project_id
        AND (p.user_id = auth.uid()
             OR (p.tenant_id IS NOT NULL AND p.tenant_id = public.get_user_tenant_id()))
    )
  );

-- INSERT: tenant-aware (user must own project)
CREATE POLICY {table}_insert_own
  ON public.{table} FOR INSERT
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM public.projects p
      WHERE p.id = {table}.project_id
        AND (p.user_id = auth.uid()
             OR (p.tenant_id IS NOT NULL AND p.tenant_id = public.get_user_tenant_id()))
    )
  );

-- UPDATE: tenant-aware
CREATE POLICY {table}_update_own
  ON public.{table} FOR UPDATE
  USING (
    EXISTS (
      SELECT 1 FROM public.projects p
      WHERE p.id = {table}.project_id
        AND (p.user_id = auth.uid()
             OR (p.tenant_id IS NOT NULL AND p.tenant_id = public.get_user_tenant_id()))
    )
  )
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM public.projects p
      WHERE p.id = {table}.project_id
        AND (p.user_id = auth.uid()
             OR (p.tenant_id IS NOT NULL AND p.tenant_id = public.get_user_tenant_id()))
    )
  );

-- Service role bypass (verify exact form against existing tables before applying)
CREATE POLICY {table}_service_role

```

### 5.3 Feature Flag Authorization

Routes `/projects/:p/feedback` and `/projects/:p/principles` check `projects.settings->>'feedback_loop_enabled'` on page load. If false/null: redirect to `/projects/:p/overview` with toast "Feedback Loop is disabled for this project. Enable it in Project Settings."

Feedback note creation is rate-limited at **30 notes per user per minute per project** (enforced in `FeedbackService.enforceRateLimit` — SELECT count from `feedback_notes` within last 60s). Exceeding this returns HTTP 429 `feedback_rate_limit_exceeded`.

### 6.7 `CorrectionsCompiler` — `_shared/feedback/compiler.ts`

**Compile input → `<corrections>` XML output.**

Budget enforcement:
- Principles: fill from priority ASC, created_at ASC; drop from tail on budget overflow (`PRINCIPLE_CHAR_BUDGET = 8,000`)
- Source corrections: all active ones (no budget cut; expected to be small)
- Synthesis corrections: fill project scope first, then artifact, then element; drop element-scope first on overflow (`CORRECTION_CHAR_BUDGET = 16,000`)

Returns `CompileOutput` with `corrections_xml`, `included_note_ids[]`, and `truncation_warning?` (includes `dropped_note_ids[]` and `reason`).

### 6.9 `VerificationPass` — `_shared/feedback/verification.ts`

**Deterministic post-generation check.** No LLM calls. Pure function.

For each synthesis feedback note in scope:
1. Find target elements by `target_identity_id` (element scope) or all elements (project/artifact scope)
2. If targets empty:
   - Self-reported not-applied → `self_reported_not_applied_accepted` (low)
   - Otherwise → `target_missing` (high)
3. If `wrong_content` set: check Jaccard similarity of wrong tokens against each target. If similarity > 0.70 → `wrong_still_present` (high)
4. If `correct_content` set: check keyword coverage of correct tokens in targets. If coverage < 0.50 → `correct_not_reflected` (medium)
5. Unverifiable (no wrong or correct, self-reported applied) → `self_reported_applied_but_unverifiable` (low)
6. Otherwise → `ok` (low)

**Overall severity:** `high` if any check is high; `medium` if any is medium; `low` otherwise.

**Application status mapping:**

| VerificationCheck.status | FeedbackApplicationStatus |
|---|---|
| `ok` | `applied_ok` |
| `self_reported_not_applied_accepted` | Derived from model's reason string; defaults to `not_applied_other` |
| `self_reported_applied_but_unverifiable` | `applied_partial` |
| `correct_not_reflected` | `applied_partial` |
| `wrong_still_present` | `not_applied_other` (sets `requires_user_review=true`) |
| `target_missing` | `not_applied_target_missing` |

### 6.10 `applySourceCorrections` — `_shared/sources/apply_corrections.ts`

**Read-time source correction. Does NOT mutate any DB row.**

- Transcript segments: replacement only in segments whose `[start,end]` overlaps correction's `[range_start_seconds, range_end_seconds]`. NULL range = all segments.
- Document text / URL extractions: whole-text replacement.
- `match_mode` semantics: whole-word CI (default), exact CI, regex (invalid regex silently skipped).
- Corrections applied in `created_at ASC` order (deterministic).

**Integration points — where to call `applySourceCorrections`:**

| Existing job stage | Integration point |
|---|---|
| `synthesis`, `meta_synthesis`, `group_synthesis` | Wrap source-loading helper; every read of transcript segments or document text goes through correction |
| `document_extraction` | At read time when `input_sources.text_content` is fetched |
| `url_extraction` | Same read-time pattern |
| `transcription` post-processing | At read time when transcript is loaded for downstream jobs |

**Implementation strategy:** Introduce `_shared/sources/loader.ts` — `loadSourceContent(input_source_id, client)` — that fetches raw content and pipes through `applySourceCorrections`. All job stages use this helper instead of direct DB reads.

### 6.12 `RegenPipeline` — `_shared/regen/pipeline.ts`

**Orchestration function `runRegenPipeline(input, userScopedClient, serviceRoleClient)`.**

15-step pipeline with two transaction windows:

**Window A (step 10) — per-artifact write:** uses existing `writeSpecDoc` / `writeTicket` / `writeGuide` / `writeKb` functions. Each is already atomic.

**Window B (steps 11-15) — AFL aux commit:** single call to `afl_commit_regen_aux` RPC (§2.15). Idempotent on retry.

**Steps:**
1. Load KB (`loadKb()` callback)
2. Load prior artifact (`loadPriorArtifact()` callback)
3. Get prior registry (`IdentityService.getPriorRegistry`)
4. Scope-resolve feedback (`FeedbackService.resolveForRegeneration`) — re-resolves at run time to catch notes superseded between enqueue and run; logs `regen.snapshot_drift` if set differs
5. Compile corrections XML (`CorrectionsCompiler.compile`)
6. Invoke generator (`input.invokeGenerator(args)` callback) — generator receives `corrections_xml` + `prior_registry`
7. Parse trailers (`parseGeneratorOutput`)
8. Reconcile (`IdentityService.reconcile`)
9. Run verification pass (`VerificationPass.run`)
10. Write new artifact (`input.writeNewArtifact(args)` callback) — Window A
11. Commit reconciliation (`IdentityService.commitReconciliation`)
12. Compose application records (`composeApplicationRecords`)
13. Insert application records + suspicions + mark notes applied via `afl_commit_regen_aux` RPC — Window B
14. Return `RegenPipelineResult`

**Cancellation:** worker polls `processing_jobs.status` between steps 4, 6, 8, 10. If `cancelled`: abort, write no artifact/AFL rows, mark job `cancelled`.

**In-flight feedback (enqueued after snaphot):** Detected in step 4. Excluded from current run for predictability. Will apply on next regen. Logged as `regen.snapshot_drift` event.

## 7. Error Handling & Edge Cases

### 7.1 Capture-Side Failures

| Failure | HTTP | Code | Recovery |
|---|---|---|---|
| CHECK constraint violation (wrong_content missing for correction) | 400 | `invalid_feedback_content` | Client fixes input, re-submits |
| RLS denied (cross-project) | 403 | `rls_denied` | Toast: "Project access denied." No recovery |
| Rate limit exceeded | 429 | `feedback_rate_limit_exceeded` | "Too many feedback notes. Try again in a minute." |
| Supersede race (old note already superseded) | 409 | `cannot_supersede_inactive_note` | UI refetches; user creates fresh note |
| source_claim without source_input_source_id | 400 | `source_claim_missing_source` | Show source picker |
| range_end < range_start | 400 | `invalid_feedback_content` | Client validates slider |
| Content > character limit | 400 | `invalid_feedback_content` | Dialog char counter prevents submit |

### 7.2 Identity-Layer Failures

| Failure | Behavior |
|---|---|
| Concurrent mintIdentity for same element | Both callers converge on same identity (unique constraint + SELECT fallback) |
| KB fact slug collision | Collision suffix applied deterministically |
| Content snapshot > 4KB | Truncated to 4,000 chars in `LEFT()` — Jaccard computed on truncated content (acceptable) |

### 7.3 Regeneration Pipeline Failures

| Failure | Status | User-visible effect | Recovery |
|---|---|---|---|
| Enqueue race (already in-flight) | 409 `regeneration_already_in_flight` | Banner: "Regeneration already running. [View progress]" | Show existing job |
| Project archived | 409 `project_archived_blocks_regen` | Toast "Project is archived." | User unarchives |
| Feature flag off | 409 `feedback_loop_disabled_for_project` | Toast | User enables in settings |
| LLM timeout | `status='retrying'` | UI shows "retrying..." | Existing retry infra; terminal → `failed` |
| Malformed XML output | `status='retrying'`/`'failed'` after max_attempts | Error banner + [Retry] | User retries or reduces corrections |
| Trailers entirely missing | All corrections get `applied_partial` | Diff banner: "Model did not report correction application; verification ran deterministically." | User reviews |
| Wrong still present (LLM ignores correction) | `not_applied_other`, `requires_user_review=true` | Red banner + [Retry stricter] | Retry or edit manually |
| writeNewArtifact fails (Window A) | No artifact/AFL rows written | Failure banner | Existing retry |
| `afl_commit_regen_aux` fails (Window B) | Partial writes possible but idempotent | Failure banner | Retry (idempotent RPC) |
| User cancels | `status='cancelled'` | "Regeneration cancelled." | Start fresh regen |
| Budget truncation | N notes dropped | Yellow info banner with dropped IDs | Promote to principles, then retry |

**Recovery path for "model ignores correction" (Retry Stricter):**
When `requires_user_review=true`: UI red banner shows "1 correction was not successfully applied." Click `[Retry stricter]` → new job enqueued with `input_payload.strict_mode=true` → `PromptWrapper` adds: "You MUST NOT reintroduce [wrong_content] anywhere in this regeneration." → verification pass retests on next completion.

### 7.4 Source Correction Failures

| Failure | Behavior |
|---|---|
| Invalid regex pattern | Skip this correction; receipt shows `dropped_reason='invalid_regex'`; UI shows "1 source correction could not be applied" |
| No segments in transcript | receipt shows `matches_applied=0`; no-op |
| Range outside all segments | receipt shows `matches_applied=0`; logged; user can edit range |
| `input_source_id` FK deleted | `source_corrections.is_active` set false on cascade; orphan suspicion raised on next regen |

### 7.5 Promotion and Reconciliation Failures

| Failure | Code | Recovery |
|---|---|---|
| Promoting non-resolved notes | `principle_requires_resolved_notes` | UI only offers promotion on resolved notes |
| Duplicate principle statement | `principle_already_exists` | Auto-link note to existing principle; toast explaining |
| Suspicion already resolved (race) | `suspicion_already_resolved` | UI refetches state |

### 7.6 Feature Flag Edge Cases

| Case | Behavior |
|---|---|
| Disable flag mid-session while notes exist | Existing notes preserved in DB; all anchors go `disabled=true`; feedback routes redirect |
| User disables flag then regenerates | Worker runs standard (non-AFL) regen; no corrections applied |

### 7.7 Form Validation (Feedback Dialog)

**Every field, every rule:**

| Field | Rule | Error message | Timing |
|---|---|---|---|
| `wrong_content` | Required for `correction\|rejection` | "Please describe what's wrong." | On submit |
| `wrong_content` | Max 4,000 chars | "Too long (max 4,000 characters)." | On change (counter) + submit |
| `correct_content` | Required for `correction\|addition` | "Please describe what it should be." | On submit |
| `correct_content` | Max 4,000 chars | "Too long (max 4,000 characters)." | On change + submit |
| `reasoning` | Required for `clarification` | "Please explain why." | On submit |
| `reasoning` | Max 2,000 chars | "Too long (max 2,000 characters)." | On change + submit |
| Source range | `range_end >= range_start` | "End must be after start." | On slider change |
