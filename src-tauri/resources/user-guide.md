### Chapter 1: First Launch

CodeMantis greets you with a Welcome Screen the first time you open the app. It checks that your system is ready, introduces the app, and offers three ways to get started.

#### What You See

The Welcome Screen fills the entire window. At the top center, the CodeMantis logo (a rounded icon) is displayed, followed by the heading **"Welcome to CodeMantis"**, the tagline *"Native desktop UI for Claude Code"*, and the current app version (e.g., `v1.0.0`).

Below the heading is a brief description:

> **CodeMantis: The AI Coding Studio for the Rest of Us.**
> You have the vision; now you have the environment to build it. CodeMantis brings the power of Claude Code into a beautiful, native macOS interface designed for the "vibe coding" era.

It highlights three selling points:
- **Visual Session Management** — Organize your thoughts and threads effortlessly.
- **Your Ideas to AI-powered Specifications to Claude Code** — Integrated to work.
- **Real-time Activity Tracking** — See exactly what your AI is doing as it happens.

The screen is divided into two side-by-side cards:

**Requirements card (left):**
A checklist of prerequisites, each with a green checkmark or an empty circle:
1. **Claude Code CLI** — Shows "Installed (vX.X.X)" when found, or "Not installed" with the install command `npm install -g @anthropic-ai/claude-code`.
2. **Authentication** — Shows "Logged in at Claude Code" when authenticated, or "Not authenticated" with the command `claude login`.
3. **You are cool and motivated** — Always checked. Ready to build something awesome.

A **Re-check** button (with a refresh icon) sits in the top-right corner of this card for re-verifying prerequisites.

**First steps card (right):**
Three clickable rows, each disabled until all prerequisites are met:
1. **Add AI API Keys** (key icon) — Labeled "Optional". Opens Settings for configuring API keys for multi-AI assistant and changelog features.
2. **Open a Project** (folder icon) — "Open an existing folder." Opens the folder picker.
3. **Create New Project** (plus icon) — "Scaffold from a template." Opens the template picker.

**Claude Code not found warning:**
If the Claude Code CLI is not detected, a yellow-bordered info box appears above the description with an alert triangle icon. It explains that CodeMantis needs Claude Code installed to work and provides two actions:
- **Get Claude Code** — A link that opens the Claude Code product page in your browser.
- **Locate Claude Code** — A button to manually browse for the Claude Code binary on your system.

At the bottom of the screen:
- A **"Do not show this again"** checkbox (checked by default).
- A **"Skip for now"** link (only visible when all prerequisites are met) to dismiss the Welcome Screen without choosing a first step.

#### How to Open / Access

- **Automatically** on first launch (when onboarding has not been completed).
- **Later:** Settings → General → the Welcome Screen can be re-triggered by resetting the onboarding flag. The setting `onboardingCompleted` in the settings store controls this.

#### User Actions

**Check prerequisites**
Click the **Re-check** button → The app re-runs Claude Code detection and authentication checks. The button shows a spinning icon while checking and reads "Checking..." during the process.

**Install Claude Code (if not found)**
Click **Get Claude Code** → Opens `https://claude.com/product/claude-code` in your default browser.
Or click **Locate Claude Code** → Opens a file picker so you can manually point CodeMantis to the `claude` binary.

**Add AI API Keys**
Click the **Add AI API Keys** row → Opens the Settings modal. This is optional — Claude Code itself does not need an API key (it uses your Claude Pro/Max subscription), but the multi-AI assistants and AI changelog features require keys from OpenAI, Google Gemini, or Anthropic.

**Open an existing project**
Click the **Open a Project** row → Opens the Project Picker with the "Open Folder" tab selected.

**Create a new project from a template**
Click the **Create New Project** row → Opens the Project Picker with the "Templates" tab selected.

**Dismiss the Welcome Screen**
Click **Skip for now** (at the bottom-right) → Closes the Welcome Screen. If "Do not show this again" is checked, the screen will not appear on future launches.

#### States

- **Default (prerequisites met):** All three first-step rows are enabled with full opacity. The "Skip for now" link is visible.
- **Default (prerequisites not met):** The three first-step rows are dimmed (50% opacity) and show a "not-allowed" cursor. Only the Requirements card and the Claude Code warning are interactive.
- **Rechecking:** The Re-check button shows a spinning refresh icon and reads "Checking..."

#### Configuration

| Setting | Location | Description |
|---------|----------|-------------|
| Do not show this again | Welcome Screen checkbox | Controls whether the Welcome Screen appears on startup |
| Claude Code binary override | Settings → General (via "Locate Claude Code") | Custom path to the `claude` binary |

#### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| (none) | The Welcome Screen has no dedicated shortcuts — interact via mouse |

#### Tips

1. **You don't need API keys to use Claude Code.** The Welcome Screen marks them as "Optional." API keys are only needed for the multi-AI assistant panel, AI changelog, and SpecWriter features.
2. **If Claude Code isn't found**, use the **Locate Claude Code** button to point CodeMantis to a non-standard installation path. This is saved as `claudeBinaryOverride` in your settings.
3. **The "Do not show this again" checkbox is checked by default.** If you want to see the Welcome Screen again later, you can re-enable it through Settings.

### Chapter 2: The CodeMantis Interface

CodeMantis uses a three-panel layout with a title bar at the top and a session sub-tab bar below it. Everything you need is accessible from this single window.

#### What You See

The interface is organized vertically and horizontally:

**Title Bar (top strip, 48px tall):**
From left to right:
- **Traffic light buttons** — The standard macOS window controls (close, minimize, maximize) in the far-left corner, with a 78px spacer.
- **Project tabs** — One tab per open project, showing the folder name. Active tab has a top accent border. Each tab shows a folder icon (when idle), a pulsing green dot (when Claude is working), or a yellow dot (when a session is stale/unresponsive). If more than one session exists in a project, a session count badge appears. Hovering or selecting a tab reveals a close (×) button.
- **Action buttons** (right side of the title bar, left to right):
  1. **New Project** (+) — Opens the template picker. Tooltip: "New project from template (Cmd+Shift+N)"
  2. **Open Project** (folder icon) — Opens the project picker. Tooltip: "Open existing project (Cmd+O)"
  3. **SpecWriter** (pen icon) — Toggles the SpecWriter slide-over. Tooltip: "SpecWriter (Cmd+Shift+B)". Shows a SpecWriter badge when a spec is in progress.
  4. **Run Application** (globe icon) — Launches or focuses the preview browser window. Tooltip: "Run Application"
  5. **Screenshot** (camera icon) — *Only visible when the preview window is open.* Captures a screenshot of the preview and adds it as an attachment to your chat. Tooltip: "Screenshot preview to chat"
  6. **MCP Servers** (blocks icon) — Opens the MCP server management modal. Tooltip: "MCP Servers (Cmd+Shift+M)"
  7. **Settings** (gear icon) — Opens the Settings modal. Tooltip: "Settings (Cmd+,)"

When no projects are open, the title bar shows the text "CodeMantis" in the center where project tabs would normally appear.

**Session Sub-Tabs (below the title bar, 32px tall):**
A row of tabs for the sessions within the currently active project:
- Each **session tab** shows: a status dot (green = idle, yellow = streaming/working), a model badge (e.g., "Sonnet"), and the session name. Active tab has a bottom accent border and elevated background.
- **"+" button** — Creates a new session in the current project. Tooltip: "New session in this project"
- **History tab** (right side, clock icon) — Shows previously closed sessions that can be resumed.
- **Project Log tab** (right side, scroll icon) — Shows all changelog entries for this project.

**Three-Panel Layout (below the sub-tabs):**
1. **Sidebar (left panel):** Shows the file tree with a "Files" header, new file/folder buttons, and a refresh button. Below the file tree: a Git Status card (if the project is a git repo) and a Context Meter showing how much of Claude's context window is used.
2. **Center panel:** The Chat Panel (conversation with Claude) with an Input Area at the bottom. When the History or Project Log tab is selected, the center panel shows that view instead of the chat.
3. **Right panel:** Tabbed panel with Activity, Terminal, Files, Changelog, and Assistant tabs.

**Resize handles** between panels allow dragging to adjust widths. The center column has a minimum width of 300px. Handles highlight in accent color when hovered or dragged.

#### How to Open / Access

The main interface appears automatically after the Welcome Screen is dismissed, or immediately on launch if onboarding was previously completed.

#### User Actions

**Switch between open projects**
Click a **project tab** in the title bar → The sidebar, chat, and right panel all update to show that project's files, sessions, and activity.

**Close a project**
Click the **×** button on a project tab (visible on hover or when the tab is active) → A confirmation dialog appears showing the project name and how many sessions will be closed.

**Resize panels**
Drag the thin divider line between any two panels → The panels resize. The divider highlights in accent color while dragging. The center panel cannot shrink below 300px.

**Open Settings**
Click the gear icon in the title bar or press ⌘, → The Settings modal opens.

**Open MCP Servers**
Click the blocks icon in the title bar or press ⌘⇧M → The MCP Server management modal opens.

**Open SpecWriter**
Click the pen icon in the title bar or press ⌘⇧B → The SpecWriter slide-over panel opens.

**Run the preview**
Click the globe icon in the title bar → Starts the dev server (if not running) or focuses the preview window (if already open).

**Screenshot the preview**
Click the camera icon (only visible when preview is open) → Captures a screenshot and adds it as an attachment to the current chat session. A toast notification confirms: "Screenshot added to chat."

#### States

- **No project open:** The sidebar shows "No project open." The center panel shows "Welcome to CodeMantis — Open a project to start a session."
- **Project open, no messages:** The center panel shows a faded CodeMantis icon with the text "Send a message to start the conversation."
- **Active session:** The session sub-tab shows a green dot (idle) or yellow dot (streaming/working).

#### Configuration

| Setting | Location | Description |
|---------|----------|-------------|
| Theme | Settings → General | Changes the color scheme of the entire interface |
| Font size | Settings → General | Adjustable via ⌘= / ⌘- / ⌘0 |

#### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| ⌘ , | Open Settings |
| ⌘ ⇧ N | New project from template |
| ⌘ O | Open existing project |
| ⌘ ⇧ M | MCP Servers |
| ⌘ ⇧ B | Toggle SpecWriter |
| ⌘ ⇧ P | Toggle Preview Window |
| ⌘ B | Toggle sidebar |
| ⌘ = | Zoom in (increase font size) |
| ⌘ - | Zoom out (decrease font size) |
| ⌘ 0 | Reset zoom |

#### Tips

1. **Drag the panel dividers** to create your ideal layout. Give more space to the chat when writing long prompts, or expand the right panel when reviewing file changes.
2. **The title bar is draggable.** Click and drag anywhere on the title bar background (not on buttons or tabs) to move the window.
3. **Use the camera button** to screenshot your running app and paste it directly into your Claude conversation — great for asking Claude to fix visual bugs.

### Chapter 3: Opening & Managing Projects

A "project" in CodeMantis is simply a folder on your computer that contains code. You can have multiple projects open simultaneously, each with its own set of sessions.

#### What You See

**Project Picker modal:**
A centered dialog (680px wide, up to 600px tall) with three tabs at the top:
- **Templates** — Browse and scaffold new projects from templates (see Chapter 15).
- **Open Folder** — Select an existing folder on your Mac.
- **Recent** — A list of previously opened projects with folder names and full paths.

The active tab is highlighted in accent color. The "Recent" tab shows a count of recent projects in parentheses.

**Open Folder tab:**
- A large dashed-border button labeled "Select a project folder..." — click to open the macOS folder picker. Once a folder is selected, it shows the folder name and full path.
- An **"Open Project"** button below — enabled once a folder is selected. Shows "Starting..." while the project is loading.

**Recent tab:**
- A list of previously opened folders, each showing the folder name (bold) and full path (dimmed) with a folder icon. Click any entry to open it immediately.
- If empty, shows "No recent projects."

**Project tabs in the title bar:**
Each open project appears as a tab showing:
- A folder icon (when idle), a pulsing green dot (when Claude is busy), or a solid yellow dot (when a session is stale — no events for 30+ seconds).
- The project folder name (truncated if too long).
- A session count badge (only if the project has more than one session).
- A close button (×) — visible on hover or when the tab is active.

#### How to Open / Access

- **Project Picker:** Press ⌘O or click the folder icon in the title bar. Also accessible from the Welcome Screen.
- **Template Picker:** Press ⌘⇧N or click the + icon in the title bar.
- **Project tabs:** Always visible in the title bar when at least one project is open.

#### User Actions

**Open an existing project**
Press ⌘O or click the folder icon → The Project Picker opens on the "Open Folder" tab. Click "Select a project folder..." → A macOS folder picker dialog appears titled "Select Project Folder." Choose a folder and click Open → The folder path appears in the picker. Click **"Open Project"** → The project opens with a new Claude Code session, and a project tab appears in the title bar.

**Open a recent project**
Press ⌘O → Switch to the "Recent" tab → Click a project from the list → It opens immediately.

**Create a new project from a template**
Press ⌘⇧N or click the + icon → The Project Picker opens on the "Templates" tab → Browse and select a template (see Chapter 15 for details).

**Switch between open projects**
Click a project tab in the title bar → The entire interface switches to that project's files, sessions, and activity.

**Close a project**
Click the × button on a project tab → A confirmation dialog appears: "Close {project name}? This will close {N} session(s) in this project." Click **Close** to confirm or **Cancel** to keep it open.

#### States

- **No projects open:** The title bar shows "CodeMantis" text where tabs would be. The sidebar shows "No project open."
- **One or more projects open:** Project tabs appear in the title bar. The active project tab has an accent-colored top border and elevated background.
- **Project busy:** The project tab shows a pulsing green dot instead of the folder icon.
- **Project stale:** If no events arrive from Claude for 30+ seconds, the dot turns solid yellow — indicating the session may be unresponsive.
- **Opening a project:** The "Open Project" button shows "Starting..." and is disabled during setup.
- **Error:** If opening fails, a red error message appears below the "Open Project" button.

#### Configuration

Recent projects are stored in browser local storage (managed by `recent-projects` utility) and persist across app restarts.

#### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| ⌘ O | Open the Project Picker (Open Folder tab) |
| ⌘ ⇧ N | Open the Project Picker (Templates tab) |

#### Tips

1. **You can have multiple projects open at once.** Each project gets its own tab in the title bar and its own set of sessions — perfect for working across related repos.
2. **Recent projects** save you time. Every project you open is remembered, so you can reopen it with two clicks.
3. **Watch the project tab dot** for quick status. Green pulsing = Claude is actively working. Yellow = something may be stuck. Folder icon = everything idle.

### Chapter 4: Chat Panel — Conversations with Claude

The Chat Panel is the center of CodeMantis. It shows your conversation with Claude Code — your messages, Claude's responses, thinking indicators, and turn statistics. All tool operations (file edits, bash commands, file reads) are routed to the Activity Feed in the right panel, keeping the chat focused on conversation text.

#### What You See

The Chat Panel occupies the center column of the three-panel layout. Messages appear in a scrollable area with a maximum content width of 720px, centered horizontally.

**User messages** appear right-aligned in accent-colored bubbles (rounded corners with a flat bottom-right corner). The message text is displayed in white-on-accent styling. A timestamp (e.g., "2:34 PM") appears below each message. A copy button appears to the left of the bubble on hover.

**Assistant messages** appear left-aligned with no bubble background. The content is rendered as Markdown with full support for:
- Headings, bold, italic, lists, tables (via GitHub Flavored Markdown)
- Code blocks with syntax highlighting, a language label, and a **Copy** button
- Inline code with monospace styling

Below each completed assistant message:
- **Turn stats button** — A small button showing the total token count and cost (e.g., "1.2K tokens $0.02"). Click it to open a detailed popover (see below).
- **Timestamp** — The time the response was received (e.g., "2:35 PM").
- **Duration** — How long the response took (e.g., "took 12s").

**Activity Chips** appear above assistant messages when Claude performed tool operations during that turn. These are compact badges that link to the Activity Feed for details.

**Streaming cursor:** While Claude is actively writing a response, a thin blinking accent-colored cursor appears at the end of the text.

**Thinking Indicator:** When Claude is busy but not yet streaming text (e.g., reading files, planning), a pill-shaped indicator appears at the bottom of the chat (pinned outside the scroll area for guaranteed visibility):
- Three animated accent-colored dots
- An activity label (e.g., "Reading file...", "Editing settings.ts", "Running command...", "Compacting context...")
- An elapsed timer showing how long Claude has been working (e.g., "1m 23s")

**Sub-agent panel:** When Claude launches sub-agents (parallel workers), an expandable panel appears below the thinking indicator:
- Shows the count (e.g., "3 sub-agents running") with total token usage
- Expandable to reveal each agent: its description, type badge, tool count, token count, and elapsed time
- Each agent has a colored status dot (yellow = preparing, green = running)
- Auto-expands when 3 or fewer agents; collapses with a toggle arrow when more

**Trivia cards:** If trivia is enabled in Settings, fun fact cards appear below the thinking indicator after a 3-second delay. Each card shows a topic badge (e.g., "SCIENCE", "HISTORY"), a fact, and a "Disable trivia" button. Special "easter egg" facts have a gold border.

**Session Status Bar:** A compact bar pinned at the very bottom of the chat panel showing:
- **Left side:** Status dot + label (Idle / Busy / Compacting), elapsed timer when busy, and current activity detail (e.g., "Editing App.tsx", "3 agents")
- **Right side:** Mode icon (shield for auto-accept, map for plan), model name (e.g., "Sonnet"), turn count, rate limit utilization (shown when > 50%, yellow when > 80%), total session tokens, total session cost, and context usage percentage (yellow at 70%, red at 90%)

**Scroll to bottom button:** When you scroll up in the chat, a floating "New messages" button with a down arrow appears at the bottom center. Click it to smoothly scroll to the latest message.

**Empty state:** When a session has no messages yet, the center shows a faded CodeMantis icon with the text "Send a message to start the conversation."

**No session state:** When no project is open, shows "Welcome to CodeMantis — Open a project to start a session."

#### How to Open / Access

The Chat Panel is always visible in the center column when a session tab is active. It is replaced by the History view or Project Log view when those tabs are selected.

#### User Actions

**Read Claude's response**
Claude's text streams in word by word with a blinking cursor. Markdown is rendered in real time as it arrives.

**Copy a user message**
Hover over your message → A copy button (clipboard icon) appears to its left → Click to copy the message text. The icon changes to a checkmark briefly.

**Copy a code block**
In any code block, click the **Copy** button in the top-right corner → The code is copied to your clipboard. The button text changes to "Copied" with a green checkmark for 2 seconds.

**View turn statistics**
After Claude finishes a response, click the small token/cost badge below the message → A popover opens showing:
- Duration (total and API time)
- Number of API calls
- Cost in USD
- Token breakdown: input, output, cache read, cache write, total

**Restart a crashed session**
If a session crashes, the last assistant message shows a blue **"Restart Session"** button with a refresh icon → Click to start a new Claude Code process for this project.

**Retry after an API error**
If an API error occurs, the error message shows a blue **"Retry"** button → Click to retry the failed request.

**Scroll to latest messages**
Scroll up in the chat, then click the floating **"New messages"** button at the bottom → Smoothly scrolls to the bottom. The button automatically disappears when you're already at the bottom.

**Expand the thinking indicator**
When sub-agents are running, click the summary header (e.g., "3 sub-agents running") → The panel toggles between collapsed and expanded views showing individual agent details.

**Disable trivia**
On any trivia card, click **"Disable trivia"** → Trivia is turned off immediately (setting persisted).

#### States

- **Default (empty):** Faded CodeMantis icon + "Send a message to start the conversation"
- **Streaming:** Claude's text appears word by word with a blinking cursor. The status bar shows "Busy" with elapsed time.
- **Thinking (busy, not streaming):** The thinking indicator appears with animated dots. Activity label updates in real time as Claude switches between tools.
- **Compacting:** When Claude's context is being compacted, the thinking indicator shows "Compacting context..." and the status bar shows "Compacting" with a yellow dot.
- **Idle:** The status bar shows "Idle" with a gray dot. No thinking indicator.
- **Error (crash):** A message with a "Restart Session" button.
- **Error (API):** A message with a "Retry" button.
- **Rate limited:** The status bar shows "RL XX%" — percentage of rate limit used.

#### Configuration

| Setting | Location | Description |
|---------|----------|-------------|
| Trivia cards | Settings → General → Trivia Cards | Enable/disable fun facts while waiting |
| Font size | Settings → General | Affects chat text size (also via ⌘= / ⌘-) |
| Default context window | Settings → General | The assumed context window size for the Context Meter |

#### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| Escape | Interrupt Claude (stop generation) when a session is busy |
| ⌘ = | Zoom in (larger text) |
| ⌘ - | Zoom out (smaller text) |
| ⌘ 0 | Reset zoom |

#### Tips

1. **The chat shows only conversation text.** All file operations, edits, and command executions appear in the Activity Feed (right panel). This keeps the chat clean and readable.
2. **Watch the Session Status Bar** at the bottom for a quick glance at model, tokens, cost, and context usage — without interrupting your flow.
3. **Press Escape to stop Claude** at any time. This interrupts the current generation immediately.

### Chapter 5: The Input Area

The Input Area sits at the bottom of the center panel. It's where you type messages to Claude, attach files, select your session mode and model, trigger slash commands, and control generation.

#### What You See

The Input Area is a rounded container with an elevated background, positioned below the chat panel. It contains:

**Text input (main area):**
A multi-line textarea (3 rows by default, grows up to 8 rows as you type). Placeholder text reads:
- "Ask Claude anything... (⌘+Enter to send)" when idle
- "Ask Claude anything... even while Claude is busy! (⌘+Enter to send)" when Claude is working
- "Open a project to start..." when no project is open

**Attachment Bar (above the textarea):**
When files are attached, thumbnail cards appear above the text input. Each card shows:
- An image thumbnail (36×36px) or a file icon
- The file name (truncated to 120px)
- The file size (e.g., "1.2 MB")
- A remove button (×) on hover

Clicking an image attachment opens a full-screen preview modal. Clicking a non-image attachment opens it in your default OS application.

**Action buttons (below the textarea):**
Left side:
- **+ File** — Opens a file dialog to attach images, documents, or code files.
- **@ Agent** — Agent mention button (placeholder for future functionality).
- **/ Cmd** — Opens the command palette (same as typing "/" in the input).

Right side:
- **Mode Selector** — Shows the current mode with an icon: Shield (Normal), ShieldCheck (Auto-Accept, green), or Map (Plan, yellow). Click to open the mode picker.
- **Model Selector** — Shows the current model name (e.g., "Sonnet") with a dropdown chevron. Click to switch models.
- **Thinking effort** — Shows 1-3 accent-colored bars with a label ("Low", "Medium", "High"). Click to open Settings.
- **Send / Stop button:**
  - When idle: A purple **Send** button with the ⌘↵ shortcut hint. Disabled (grayed out) when the input is empty and no attachments are present.
  - When Claude is busy: A red **Stop** button with an "Esc" hint.

**Keyboard shortcut hints (bottom edge):**
A subtle line reading: "Shift+Tab to switch mode" and "⌘+/⌘− to adjust font size"

**Command Palette (above the input, floating):**
Appears when you type "/" at the beginning of a line. A dropdown list of available commands, each showing:
- The command name in monospace accent color (e.g., `/commit`)
- A description
- An argument hint (shown when the command is selected)
- A category badge: **Skill** (accent color), **Built-in** (dim), or **Opens CLI** (yellow)

**Drop zone:**
When dragging files over the input area, the background turns to a subtle accent tint and text reads "Drop files to attach."

#### How to Open / Access

The Input Area is always visible at the bottom of the center panel when a session is active. It is hidden when the History or Project Log view is shown.

#### User Actions

**Send a message**
Type your message → Press ⌘Enter (default shortcut) → The message is sent and appears as a user bubble in the chat. The input clears immediately.
Alternatively, click the **Send** button.

**Send with attachments**

### Chapter 6: Session Modes

Session modes control how much autonomy Claude has when performing actions. There are three modes, and you can switch between them at any time during a session.

#### What You See

The mode selector appears in the bottom-right of the Input Area, showing the current mode icon and label:
- **Normal** — Shield icon, default text color. "Ask permission before edits."
- **Auto-Accept** — ShieldCheck icon, green. "Accept all tool calls automatically."
- **Plan** — Map icon, yellow. "Plan only, no code changes."

Clicking the selector opens a popup menu with all three options. Each option shows:
- The mode icon
- The mode name (bold)
- A short description

The currently active mode is highlighted with an accent background.

The Session Status Bar at the bottom of the chat also reflects the active mode:
- **Auto-Accept:** A green shield-check icon appears in the right section.
- **Plan:** A yellow map icon appears in the right section.

#### How to Open / Access

- Click the mode selector in the Input Area action bar.
- Press ⌘. (Command + period) to cycle through modes.
- Press Shift+Tab to cycle through modes.

#### User Actions

**Switch to Normal mode**

**Switch to Auto-Accept mode**
Click the mode selector → Select **Auto-Accept** → Claude executes all tool calls automatically without asking. The mode icon turns green. This is faster but gives you less control — Claude will edit files and run commands without pausing.

**Switch to Plan mode**
Click the mode selector → Select **Plan** → Claude only reasons, plans, and discusses — it never touches code or runs commands. The mode icon turns yellow. This is ideal for architecture discussions, code reviews, or exploring approaches before committing to changes.

#### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|

| Shift+Tab | Cycle through modes (same as ⌘.) |

#### Tips

1. **Start in Plan mode** for complex tasks. Ask Claude to outline its approach first, review the plan, then switch to Normal or Auto-Accept to execute.
2. **Auto-Accept is great for trusted operations** like formatting code, writing tests, or making well-defined changes. Switch back to Normal when Claude is making decisions you want to review.
3. **Mode changes take effect immediately** — even mid-conversation. If you realize Claude is about to do something risky, switch to Normal before the next tool call.

### Chapter 7: Tool Approvals

When Claude wants to edit a file, run a command, or perform any tool operation in Normal mode, it asks for your permission through the Tool Approval modal. This is your checkpoint to review what Claude is about to do before it happens.

#### What You See

The Tool Approval modal is a centered dialog (440px wide) with a semi-transparent backdrop. It contains:

**Header:**
- A yellow shield-alert icon (in a yellow-tinted circle)
- Title: **"Approve Tool?"**
- Subtitle: "Claude wants to use a tool in **{project name}**" (the project name is shown in accent color)
- **Queue navigation** (only visible when multiple approvals are queued): Left/right chevron arrows with a counter (e.g., "2/5") showing which approval you're viewing.

- A **Tool Badge** showing the tool type with its associated color (see Chapter 9 for badge colors)

**Action buttons (bottom):**
- **Left side:** "Always allow {tool name} in this session" — Click to auto-approve all future uses of this specific tool in the current session.
- **Right side:**
  - **Approve all ({count})** — Only visible when multiple approvals are queued. Approves every pending approval at once.
  - **Deny** — Rejects the tool call. Claude receives a "Denied by user" message.
  - **Approve** — Approves the tool call. Claude proceeds with the operation.

#### How to Open / Access

The Tool Approval modal opens automatically whenever Claude requests permission to use a tool in **Normal mode**. It does not appear in Auto-Accept mode (approvals are auto-granted) or Plan mode (tool calls are restricted).

#### User Actions

**Approve a tool call**
Click **Approve** or press Enter → Claude proceeds with the operation.

**Deny a tool call**
Click **Deny** or press Escape → Claude receives a denial message and adjusts its approach.

**Approve all queued tool calls**

**Always allow a tool for this session**
Click **"Always allow {tool name} in this session"** → The tool is approved now, and all future uses of the same tool in this session are auto-approved without showing the modal. This is tracked per-session and resets when the session ends.

**Navigate between queued approvals**
When multiple approvals are pending, click the **left** (◀) or **right** (▶) chevron arrows, or press the **Left Arrow** and **Right Arrow** keys → View different pending approvals without approving or denying them.

#### States

- **Single approval:** The queue counter and navigation arrows are hidden. Just the tool details and Approve/Deny buttons.
- **Multiple approvals queued:** The queue counter (e.g., "2/5") and chevron arrows appear in the header. The "Approve all" button appears.
- **Auto-closed:** When the last approval in the queue is resolved, the modal closes automatically.
- **Auto-opened:** When a new approval arrives while no modal is showing, it opens automatically.

#### Configuration

Tool approvals are only active in Normal mode. Switch to Auto-Accept mode (see Chapter 6) to bypass approvals entirely.

The "Always allow" setting is per-session and per-tool — it does not persist across sessions or affect other tool types.

#### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| Enter | Approve the current tool call |
| Escape | Deny the current tool call |
| ← (Left Arrow) | View previous approval in queue |
| → (Right Arrow) | View next approval in queue |
| ⌘ A | Approve all queued tool calls |

#### Tips

1. **Review the JSON preview carefully.** It shows exactly what Claude intends to do — the file path, the edit content, or the bash command. This is your last chance to catch mistakes before they happen.
2. **Use "Always allow" for repetitive tools.** If Claude is reading many files during research, click "Always allow Read in this session" to speed things up without losing control over writes and edits.
3. **Press Enter/Escape for fast approvals.** You don't need to reach for the mouse — Enter approves, Escape denies.

**Important:** Selecting a CLI-only command in CodeMantis opens the Claude Code CLI overlay, but it does **not** automatically send the command to Claude Code. You need to **re-enter the slash command** (e.g., `/model`, `/doctor`) inside the Claude Code terminal that appears. When you're done, press **Escape** to close the overlay and return to CodeMantis — your session resumes right where you left off.

**CLI Overlay Lifecycle**
The CLI Overlay works as follows:
1. You select a CLI-only command (or press ⌘/).
2. CodeMantis **pauses** the current stream-json session.
3. A PTY terminal opens with `claude --resume {session_id}` in interactive mode.
4. **You re-enter the slash command** inside the Claude Code CLI (the command is not forwarded automatically).
5. You interact directly with Claude Code's interactive CLI (the same interface you'd get by running `claude` in a terminal).
6. When you're done, press **Escape** or click the × button — the overlay closes, the PTY is killed.
7. The stream-json session **resumes automatically** — no work is lost.

**Close the CLI Overlay**
Press Escape or click the × button → The overlay closes, the interactive CLI exits, and the normal stream-json session resumes.

**Use slash commands in assistant tabs**
Claude Code assistant tabs also support slash commands. Type "/" in the assistant input to open the command palette. Supported commands include `/help`, `/clear`, `/context`, `/cost`, `/exit`, and `/rename`.

**Custom Skills (User-Defined Commands)**

Skills are custom commands created by you or the project. They live in `.md` files inside specific directories:

| Location | Scope | Priority |
|----------|-------|----------|
| `{project}/.claude/commands/` | Project-level | Highest (overrides user-level) |
| `{project}/.claude/skills/` | Project-level (skill format) | High |
| `~/.claude/commands/` | User-level (all projects) | Normal |
| `~/.claude/skills/` | User-level (skill format) | Lower |

**Commands** (`.claude/commands/`) are simple `.md` files. The filename becomes the command name. Example: `.claude/commands/review.md` becomes `/review`.

**Skills** (`.claude/skills/`) are folders containing a `SKILL.md` file with frontmatter. Example: `.claude/skills/deploy/SKILL.md` becomes `/deploy`.

**Frontmatter options** (optional YAML block at the top of the file):
```yaml
---
name: custom-name          # Override the filename-derived name
description: My command    # Shown in the command palette
argument-hint: <file>      # Placeholder text for arguments
model: claude-sonnet-4     # Force a specific model
user-invocable: false      # Hide from the command palette
allowed-tools: Read, Write # Restrict which tools can be used
context: fork              # Fork the conversation context
---
```

**Template variables available in the command body:**
- `$ARGUMENTS` / `${ARGUMENTS}` — All arguments after the command name
- `$0` through `$9` — Individual positional arguments
- `${CLAUDE_SESSION_ID}` — Current CLI session ID
- `${CLAUDE_SKILL_DIR}` — Directory containing the skill file
- `` !`command` `` — Shell command expansion (executed during skill loading)

#### States

- **Command Palette loading:** "Loading commands..." while the command list is being discovered.
- **No results:** "No commands matching '/{query}'" when the search returns nothing.
- **CLI Overlay loading:** "Pausing session and starting Claude CLI..." while the overlay initializes.
- **CLI Overlay error:** "Failed to start Claude CLI" with error message.
- **CLI Overlay ready:** Full terminal visible and interactive.

#### Configuration

Skill commands are discovered automatically from:
- `.claude/commands/` in your project directory (project-level, highest priority)
- `.claude/skills/` in your project directory (project-level skill format)
- `~/.claude/commands/` in your home directory (user-level, all projects)
- `~/.claude/skills/` in your home directory (user-level skill format)

Each `.md` file in these directories becomes a slash command. The file name becomes the command name, and the file content becomes the prompt template. Project-level commands override user-level commands with the same name.

#### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| / | Open Command Palette (when typed at start of input) |
| ⌘ / | Open CLI Overlay |
| Arrow keys | Navigate Command Palette |
| Enter | Select command |
| Tab | Autocomplete command |
| Escape | Close Command Palette or CLI Overlay |

#### Tips

1. **Create custom skill commands** by adding `.md` files to `.claude/commands/` in your project. For example, `review.md` with a code review prompt becomes the `/review` command.
2. **Use `/compact` when context is running high** — it reduces token usage by compacting the conversation history, letting you continue longer without starting a new session.
3. **The CLI Overlay is temporary** — it pauses your normal session, lets you use interactive CLI features, then resumes exactly where you left off. No work is lost.
4. **Use `/doctor` as your first troubleshooting step** — it checks binary version, authentication, configuration, and MCP connectivity all at once.

### Chapter 26: Error Recovery & Troubleshooting

CodeMantis is designed to handle errors gracefully. This chapter covers every error state you might encounter and how to recover from each one.

#### What You See

##### Session Status Bar Indicators

The Session Status Bar at the bottom of the chat panel (see Chapter 4) shows several indicators that help diagnose issues:

- **Status**: "Idle" (gray dot), "Busy" (pulsing green dot), or "Compacting" (pulsing yellow dot).
- **Rate limit utilization**: "RL XX%" appears when rate limit usage exceeds 50%. Turns yellow at 80%.
- **Context percentage**: "ctx XX%" with color coding (normal, yellow at 70%, red at 90%).
- **Mode icon**: Shield (Normal), ShieldCheck (Auto-Accept), Map (Plan).

##### Confirm Close Modal

When closing a session or project, a confirmation dialog appears:

- **Close session:** Title: "Close session '{name}'?" Description: "The Claude CLI process will be stopped."
- **Close project:** Title: "Close project '{name}'?" Description: "All N sessions and their CLI processes will be stopped."
- Buttons: **Cancel** (secondary) / **Close** (red).
- Keyboard: Enter = Close, Escape = Cancel.

##### Plan Complete Modal

When a Plan mode conversation finishes, a dialog appears:

- **Title:** "Plan Complete"
- **Description:** "Claude has finished planning. Ready to implement?"
- **Auto-Accept toggle:** A checkbox labeled "Enable Auto-Accept" with description "Approve all tool calls automatically during implementation."
- **Buttons:** "Later" (secondary) / "Implement Now" (primary accent).
- Keyboard: Enter = Implement Now.

Clicking "Implement Now" sends "Go ahead, implement the plan." to Claude and optionally switches to Auto-Accept mode.

##### Error Toast Notifications

Errors throughout the app are surfaced as toast notifications — small temporary messages that appear at the top of the screen. They show the error message in red and automatically dismiss after a few seconds.

##### Session Crash Recovery

If a Claude Code session crashes:
- The last assistant message displays a **"Restart Session"** button (blue, with a refresh icon).
- Click it to start a new Claude Code process for the same project.
- Your files are untouched — only the conversation state is affected.

##### API Error Recovery

If an API error occurs during a Claude response:
- The error message displays a **"Retry"** button (blue, with a refresh icon).
- Click it to retry the failed request.

##### Rate Limiting

When you hit Claude's rate limit:
- The Session Status Bar shows "RL XX%" in yellow (at 80%+).
- Claude Code handles rate limiting internally with automatic retries and backoff.
- A countdown may appear in the chat while waiting for the rate limit to reset.

##### Stale Connection Detection

If no events arrive from Claude for 30+ seconds while a session is marked as busy:
- The project tab dot turns solid yellow (instead of pulsing green).
- This indicates the session may be unresponsive — consider closing and restarting it.

#### How to Open / Access

Error states appear automatically when issues occur. No manual navigation required.

#### User Actions

**Restart a crashed session**
Click the **"Restart Session"** button on the crash message → A new Claude CLI process starts for this project. The previous conversation is gone, but your files remain unchanged.

**Retry after an API error**
Click the **"Retry"** button on the error message → The failed request is retried.

**Handle rate limiting**
Wait for the rate limit to reset (usually 1-5 minutes). The status bar shows the current utilization. Claude Code automatically retries once the limit clears.

**Confirm or cancel closing**
In the Confirm Close dialog: Click **Close** or press Enter to confirm. Click **Cancel** or press Escape to keep the session/project open.

**Implement a completed plan**
In the Plan Complete dialog: Optionally check "Enable Auto-Accept" → Click **"Implement Now"** or press Enter. Or click **"Later"** to dismiss and implement manually.

**Recover from a stale session**
If the project tab shows a solid yellow dot for an extended period: Close the session (⌘W → confirm) and create a new one (⌘N). You can resume the conversation from the History tab (see Chapter 8).

#### States

- **Healthy:** Green pulsing dot (busy) or gray dot (idle). Normal operation.
- **Compacting:** Yellow pulsing dot. Claude is compacting context — temporary, resolves automatically.
- **Rate limited:** "RL XX%" in the status bar. Automatic retry in progress.
- **Stale:** Solid yellow dot on project tab. No events for 30+ seconds.
- **Crashed:** "Restart Session" button on the last message. Session terminated.
- **API error:** "Retry" button on the error message. Single request failed.

#### Configuration

No user-configurable error handling settings. Error behavior is built into the app.

#### Data Safety

- **Your files are never at risk.** If CodeMantis crashes or a session dies, your project files remain exactly as they were at the last edit.
- **Sessions are preserved server-side.** Closed sessions can be resumed from the History tab (see Chapter 8) because the conversation history lives on Claude's servers.
- **The database is backed up automatically.** Before the app starts, `codemantis.db` is copied to `codemantis.db.backup`.
- **App data location:** `~/Library/Application Support/dev.codemantis.app/`

#### Tips

1. **Don't panic on crashes.** Your project files are always safe. Just click "Restart Session" and continue where you left off — or resume from History.
2. **If a session feels stuck** (solid yellow dot for 30+ seconds), close it and start a new one. You can resume the previous conversation from the History tab.
3. **Rate limiting is normal** on heavy usage. The app handles it automatically with retries — just wait a moment and it will resume.

#### Skill Commands (from .claude/commands/ and .claude/skills/)

Skills are custom commands created by you or the project. They live in `.md` files:

| Location | Scope | Priority |
|----------|-------|----------|
| `{project}/.claude/commands/*.md` | Project-level | Highest (overrides user-level) |
| `{project}/.claude/skills/*/SKILL.md` | Project-level (skill format) | High |
| `~/.claude/commands/*.md` | User-level (all projects) | Normal |
| `~/.claude/skills/*/SKILL.md` | User-level (skill format) | Lower |

**Creating a skill command:**
1. Create a `.md` file in your project's `.claude/commands/` directory (e.g., `review.md`).
2. Write a prompt template in the file (e.g., "Review the code I just wrote for bugs, security issues, and performance problems.").
3. Type `/review` in the chat input → The prompt is expanded and sent to Claude.

**Frontmatter options** (optional YAML block at the top of the `.md` file):

| Option | Description | Example |
|--------|-------------|---------|
| `name` | Override the filename-derived command name | `name: custom-name` |
| `description` | Shown in the command palette | `description: My command` |
| `argument-hint` | Placeholder text for arguments | `argument-hint: <file>` |
| `model` | Force a specific model for this command | `model: claude-sonnet-4` |
| `user-invocable` | Hide from the command palette when `false` | `user-invocable: false` |
| `allowed-tools` | Restrict which tools can be used | `allowed-tools: Read, Write` |
| `context` | Fork the conversation context | `context: fork` |

**Template variables available in the command body:**

| Variable | Description |
|----------|-------------|
| `$ARGUMENTS` / `${ARGUMENTS}` | All arguments after the command name |
| `$0` through `$9` | Individual positional arguments |
| `${CLAUDE_SESSION_ID}` | Current CLI session ID |
| `${CLAUDE_SKILL_DIR}` | Directory containing the skill file |
| `` !`command` `` | Shell command expansion (executed during skill loading) |
